# Let's Get Functional

**A Practitioner's Guide to Functional Programming in Familiar Languages**

> *This is a placeholder. The introduction will be written as the book develops.*

## Audience

Working software engineers (3+ years) fluent in at least one of TypeScript, Kotlin, Swift, or Java. They ship production code in OO-dominated ecosystems and sense there's a better way but haven't had a structured path into FP that respects their existing knowledge. Not beginners, not academics — practitioners.

## Approach

- **Polyglot examples** — TypeScript, Kotlin, Swift, Java as the situation demands. No single primary language. Pick whichever makes the concept clearest.
- **Honest trade-offs** — every technique gets a "where this hurts" sidebar. No silver bullets.
- **OOP ↔ FP direct comparisons** — side-by-side code where it illuminates, not as a gimmick.
- **Progressive disclosure** — concepts build on each other. No forward references without a reason.
